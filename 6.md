```md id="l5-analysis-intelligence-v1"
# Layer: Analysis & Intelligence

---

## 1. Purpose

Provide high-value insights and decision support by analyzing the enriched knowledge graph.

This layer enables:
- Impact analysis
- Dependency analysis
- Architectural insights
- Code quality signals
- Risk detection

Transforms:
> **Searchable knowledge → actionable intelligence**

---

## 2. Scope

### 2.1 In-Scope
- Impact analysis (change propagation)
- Dependency graph traversal
- Architectural pattern validation
- Hotspot detection
- Risk and anomaly detection

### 2.2 Out-of-Scope
- Data extraction (Layer 2)
- Enrichment (Layer 3)
- Raw search (Layer 4)
- UI rendering

---

## 3. Core Agent Types

---

### 3.1 Impact Analysis Agent

#### Responsibility:
Determine what is affected by a change.

#### Capabilities:
- Traverse dependency graph
- Identify:
  - direct dependencies
  - indirect dependencies
- Estimate blast radius

---

### 3.2 Dependency Analysis Agent

#### Responsibility:
Analyze structure of dependencies.

#### Outputs:
- Dependency chains
- Cycles
- Layer violations

---

### 3.3 Architecture Analysis Agent

#### Responsibility:
Evaluate architectural patterns and deviations.

#### Capabilities:
- Detect layering issues
- Identify anti-patterns:
  - god classes
  - cyclic dependencies
  - tight coupling

---

### 3.4 Hotspot Detection Agent

#### Responsibility:
Identify critical or risky parts of the system.

#### Signals:
- High dependency count
- Centrality in graph
- Frequent usage

---

### 3.5 Risk Scoring Agent

#### Responsibility:
Assign risk scores to entities.

#### Risk Factors:
- Complexity
- Coupling
- Change impact
- Lack of test coverage (future)

---

## 4. Inputs

### 4.1 Required Inputs

| Name | Type | Description |
|------|------|-------------|
| entity_graph | object | Enriched entity + relationship graph |

---

### 4.2 Optional Inputs

| Name | Type | Description | Default |
|------|------|-------------|---------|
| target_entity | string | Focus entity for analysis | null |
| analysis_type | string | impact, dependency, risk | all |

---

### 4.3 Context Sources

- Knowledge graph (Layer 1 + 3)
- Search results (Layer 4)
- Historical metadata (optional)

---

## 5. Processing Logic

---

### Step 1: Input Validation
- Validate entity existence
- Validate graph integrity

---

### Step 2: Graph Traversal

#### For Impact Analysis:
- BFS/DFS traversal
- Depth-limited exploration

#### For Dependency Analysis:
- Identify:
  - upstream
  - downstream

---

### Step 3: Pattern Evaluation
- Apply architectural rules
- Detect violations

---

### Step 4: Metric Computation

Compute:
- degree centrality
- dependency depth
- fan-in / fan-out

---

### Step 5: Risk Scoring

Example formula:

```

risk_score =
w1 * complexity +
w2 * coupling +
w3 * impact_radius

````id="c8o5hp"

---

### Step 6: Insight Generation
- Convert metrics → human-readable insights

---

### Step 7: Output Preparation
- Structure results
- Attach explanation

---

## 6. Output Specification

  "results": {
````

## 9. Examples

### 9.1 Input

```json id="b8v3ch"
{
  "target_entity": "UserService",
  "analysis_type": "impact"
}
```

---

### 9.2 Output

```json id="bq5f5e"
{
  "entity_id": "UserService",
  "analysis_type": "impact",
  "results": {
    "affected_entities": [
      "UserController",
      "OrderService"
```

---

### 9.3 Edge Cases

* Isolated entity → low impact
* Cyclic dependency → flagged
* Large graph → depth-limited traversal

---

## 10. Failure Handling

### Failure Conditions

* Graph inconsistency
* Missing entity

### Strategy

* Return partial results
* Log inconsistencies
* Lower confidence score

---

## 11. Evaluation Criteria

### Functional

* Impact accuracy ≥ 90%
* Risk scoring consistency ≥ 85%

---

### Performance

* Analysis latency < 500ms (avg)

---

### Test Scenarios

* Highly coupled system
* Microservices architecture
* Legacy monolith

---

## 12. Constraints & Guardrails

* No hallucinated relationships
* Must rely on graph data
* Deterministic scoring rules
* Explainable insights mandatory

---

## 13. Observability

Track:

* analysis execution time
* number of entities analyzed
* risk distribution

---

## 14. Security

* Enforce access control
* Do not expose restricted entities

---

## 15. Versioning

* Analysis model versioned
* Scoring weights configurable

---

## 16. Future Enhancements

* Runtime correlation (logs + traces)
* Predictive impact analysis
* ML-based anomaly detection
* Auto-refactoring suggestions
